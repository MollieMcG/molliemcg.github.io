How it Started

An experience at a local cinema using self-service tablet kiosks sparked this project. While the ticket ordering process functioned smoothly, the seat selection interface proved problematic. The system exhibited several usability flaws, including automatic and often incorrect seat assignments, and unpredictable behavior when users attempted to modify their selections. This frustrating experience highlighted a clear opportunity for improvement and motivated the creation of this redesign.

The Problem

Recognizing the need for a user-friendly seat selection experience, I undertook this UX/UI side project to address the usability issues present in the current tablet system. This project provided an opportunity to develop a responsive solution and enhance my Figma skills.

Research

Initial User Flow

A user flow was created to map the user's journey through the seat selection process at the beginning of the project.

User Persona

Following the development of the user flow, a user persona was created to represent a frequent moviegoer who prioritizes a seamless ordering process and precise seat selection for an optimal cinematic experience. Although the project stemmed from personal experience, understanding the needs of regular moviegoers was crucial.

Gathering Inspiration

I began to pull images from online and from the website of the theater I went to and began to analyze and brainstorm improvements from the original site.

The Outcome

Undertaken early in my bootcamp, this project served as a practical exercise in applying core UX/UI principles and expanding my Figma prototyping capabilities. Future iterations will include the design of the cart screens and an investigation into prototyping the project's code.
